class fullrmc.Core.Constraint.EnhanceOnlyConstraint(engine, rejectProbability)¶

Bases: fullrmc.Core.Constraint.Constraint

An enhance only constraint is a constraint that doesn’t count into the total chiSquare of the Engine. But it’s internal chiSquare must monotonously decrease or remain the same from one engine step to another. If chiSquare of an EnhanceOnlyConstraint increases the step will be rejected even before engine’s new chiSquare get computed.

Parameters:
  1. engine (None, fullrmc.Engine): The constraint fullrmc engine.
  2. rejectProbability (None, numpy.ndarray): rejection probability numpy.array of dimension (100,). If None, rejectProbability will be automatically generated to 1 which means total rejection for all steps where chiSquare increases.

angles¶

This is a C compiled Cython generated module to calculate angles constraints. It contains the following methods.

single_angles: It calculates the angles constraint of a single atom.
Arguments:
  1. centralAtomIndex (int32): The central atom index.
  2. leftIndexes (int32 array): The centralAtom’s angles left atoms indexes.
  3. rightIndexes (int32 array): The centralAtom’s angles right atoms indexes. number of leftIndexes must be equal to number of rightIndexes.
  4. boxCoords (float32 array): The whole system box coordinates.
  5. basis (float32 array): The box vectors.
  6. lowerLimit (float32 array): The (numberOfLeftIndexes) array for lower limit or minimum bond length allowed.
  7. upperLimit (float32 array): The (numberOfLeftIndexes) array for upper limit or maximum bond length allowed.
  8. reduceAngleToUpper (bool): Whether to reduce angle found out of limits to the difference between the angle and the upper limit. When True, this flag has the higher priority. DEFAULT: False
  9. reduceAngleToLower (bool): Whether to reduce angle found out of limits to the difference between the angle and the lower limit. When True, this flag may lose its priority for reduceAngleToUpper if the later is True. DEFAULT: False
Returns:
  1. result (python dictionary): It has only two keys.
    1. angles: The calculated angles (rad).
    2. reducedAngles: The reduced angles (rad)
full_angles: It calculates the angles constraint of all atoms given a angles dictionary.
Arguments:
  1. angles (python dictionary): The angles dictionary. Where keys are central atoms indexes and values are dictionary of leftIndexes array, rightIndexes array, lowerLimit array, upperLimit array
  2. boxCoords (float32 array): The whole system box coordinates.
  3. basis (float32 array): The box vectors.
  4. reduceAngleToUpper (bool): Whether to reduce angle found out of limits to the difference between the angle and the upper limit. When True, this flag has the higher priority. DEFAULT: False
  5. reduceAngleToLower (bool): Whether to reduce angle found out of limits to the difference between the angle and the lower limit. When True, this flag may lose its priority for reduceAngleToUpper if the later is True. DEFAULT: False
Returns:
  1. result (python dictionary): where keys are central atoms indexes and values are dictionaries of exactly two keys as such.
    1. angles: The calculated bonds length
    2. reducedAngles: The reduced bonds length

bonds¶

This is a C compiled Cython generated module to calculate bonds constraints. It contains the following methods.

single_bonds: It calculates the bonds constraint of a single atom.
Arguments:
  1. atomIndex (int32): The index of the atom.
  2. bondedIndexes (int32 array): The bonded atoms indexes array.
  3. boxCoords (float32 array): The whole system box coordinates.
  4. basis (float32 array): The box vectors.

single_improper_angles: It calculates the improper angles constraint between an improper atom and a plane atoms.
The plane normal vector is calculated using the right-hand rule where thumb=ox vector, index=oy vector hence oz=normal=second finger
Arguments:
  1. improperAtomIndex (int32): The atom index that must be in plane.
  2. oAtomIndex (int32 array): The first atom index to build the plane considered as origin.
  3. xAtomIndex (int32 array): The second atom index to build the plane considered as ox vector. number of oAtomIndex must be equal to number of xAtomIndex.
  4. yAtomIndex (int32 array): The third atom index to build the plane considered as oy vector. number of xAtomIndex must be equal to number of yAtomIndex.
  5. boxCoords (float32 array): The whole system box coordinates.
  6. basis (float32 array): The box vectors.
  7. lowerLimit (float32 array): The (oAtomIndex) array for lower limit or minimum bond length allowed.
  8. upperLimit (float32 array): The (oAtomIndex) array for upper limit or maximum bond length allowed.
